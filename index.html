Introduction

Utils is a modular DOM library with the explicit objective of facilitating defensive HTML DOM scripting. It has been widely tested in browsers both new and old in order to maximize compatibility.

Utils contains zero lines of environment-specific code; this approach maximizes compatibility. Each morsel of feature testing logic is based upon substantial research of various environments.

Quick Facts

  • Standards-based

    Utils is developed with standards in mind. Most "forks" used in the API implement algorithms specified in both the DOM 4 and HTML 5 specifications.

  • Developer Friendly

    Utils is heavily documented and contains only clean code. This means that both developers and users alike can easily study the API.

  • Portable

    Utils is developed to work in a maximal amount of environments. Because documentation is provided within the script itself, developers have no need to consult the full documentation. The web site itself is built to be accessible to as many browsers as possible; mobile browsers are encouraged.

Tenets

Utils is developed with the following core tenets in mind:

  • Graceful Degradation;
  • Namespacing;
  • Modularity;
  • Minimal DOM Abstraction.

Support

Utils is developed to gracefully degrade in as many environments as possible.

The following environments have been explicitly tested and passed tests with a 100% grade (permitting sufficient DOM support to run tests):

  • Internet Explorer 6-9;
  • Firefox 1-12;
  • Opera 7-11;
  • Safari 3.1-5;
  • Chrome 1-40.

Utils has been tested in older versions of the above along with miscellaneous browsers. Support for the preceding is positive, but not qualitative.

Tests are conducted with a basic subset of the DOM Level 1 Core API. Any browser that does not support the preceding API cannot have its support qualitatively evaluated.

Style

Utils is written in a style that's subjectively easy to read and follow. No line of code exceeds 72 columns; every function's body is 251 lines of code or less; each function contains documentation for portability and clarity. Furthermore, the code is intended to be reliable and friendly; zero corners are cut.

Utils passes JSLint with only two options selected: whitespace and "use strict";.

Utils also passes JSHint with only "use strict"; disabled and no environmental assumptions.
