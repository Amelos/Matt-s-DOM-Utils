Introduction

Utils is a modular DOM library with the explicit objective of facilitating defensive HTML DOM scripting. It has been widely tested in browsers both new and old in order to maximize compatibility.

Utils contains zero lines of environment-specific code; this approach maximizes compatibility. Each morsel of feature testing logic is based upon observation(s) from various environments.

Tenets

Utils is developed with the following core tenets in mind:

  • Namespacing;
  • Modularity;
  • Gracefull Degradation;
  • Minimal DOM Abstraction.

Support

Utils is developed to gracefully degrade in as many environments as possible.

The following environments have been explicitly tested:

  • Internet Explorer 6-8;
  • Mozila Firefox 1-3;
  • Opera 3-11;
  • Safari 3.1-5.

Style

Utils is written in a style that's subjectively easy to read and follow. No line of code exceeds 72 columns; every method's body is 20 lines of code or less; each method contains documentation for portability and clarity. Furthermore, the code is intended to be reliable and friendly; zero hacks are used.

Utils passes JSLint with only two options selected: whitespace and "use strict";.

Utils passes JSHint with only "use strict"; disabled and no environmental assumptions.
